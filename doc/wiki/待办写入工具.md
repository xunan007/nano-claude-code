# 待办写入工具

## 数据结构

新增的 todo 状态只存在于当前运行进程里，用来维护本轮会话的计划。

```ts
type PlanStatus = "pending" | "in_progress" | "completed";

type PlanItem = {
    content: string; // 这一步要做什么
    status: PlanStatus; // 这一步现在处于什么状态
    activeForm: string; // 当它正在进行时的描述
};

type PlanningState = {
    items: PlanItem[];
    roundsSinceUpdate: number; // 连续过去多少轮还没有更新计划
};
```

- PlanStatus
    - `pending`：还没开始
    - `in_progress`：正在进行
    - `completed`：已经完成
- PlanItem
    - `content`：计划项内容
    - `status`：计划项状态
    - `activeForm`：可选的进行时描述，只有当前项是 `in_progress` 时渲染出来
- PlanningState
    - `items`：当前会话的计划列表
    - `roundsSinceUpdate`：连续几轮工具调用没有刷新计划，用来判断是否需要插入 reminder

## TodoManager 实现

- 模型工具调用会把计划传入
    - .update
        - 计划长度不能超过 12
        - 校验计划格式，保证最多只有一个 in_progress
        - 更新成功后重置 roundsSinceUpdate=0
        - 返回渲染后的计划文本

## Reminder 协同 Agent Loop

- runOneTurn
    - 调用 LLM 响应 tool_call
    - 如果发现用了 todo 工具，那么不需要插入提醒
    - 否则需要根据执行轮次决定是否在 message 插入一条 reminder
