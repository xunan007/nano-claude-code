# 上下文压缩

## 为什么需要压缩

s06 的目标是让 agent 在长任务里继续工作，而不是被越来越大的历史消息拖垮。

这里实现了三层压缩：

- 大工具输出落盘：完整结果写到 .task_outputs/tool-results，上下文里只保留预览
- 旧工具结果微压缩：只保留最近几个工具结果，旧结果替换成短占位符
- 全量历史压缩：当消息历史超过阈值，先保存 transcript，再让模型总结当前工作状态

## CompactState

压缩状态只存在于当前进程：

```ts
type CompactState = {
    hasCompacted: boolean;
    lastSummary: string;
    recentFiles: string[];
};
```

- hasCompacted：是否已经发生过一次完整压缩
- lastSummary：最近一次完整压缩得到的摘要
- recentFiles：最近读取过的文件，压缩后提示模型必要时重新打开

## 三种压缩说明

### 大输出落盘

bash 和 read_file 会拿到当前 tool call id。如果工具输出超过阈值，会写入：

```txt
.task_outputs/tool-results/<tool_use_id>.txt
```

上下文里只保留：

```txt
<persisted-output>
Full output saved to: .task_outputs/tool-results/call_xxx.txt
Preview:
...
</persisted-output>
```

这样模型还能看到前几千字符判断结果类型，但不会把整段大输出塞进活跃上下文。

### 微压缩

每轮模型调用前会执行 microCompact(messages)：

- 收集历史里的 tool_result
- 保留最近 3 个工具结果
- 更早且较长的结果替换成：

```txt
[Earlier tool result compacted. Re-run the tool if you need full detail.]
```

这一步不需要调用模型，成本很低。

### 完整压缩

如果 JSON.stringify(messages).length 超过 CONTEXT_LIMIT，会触发自动压缩：

```txt
[auto compact]
```

完整历史先保存到：

```txt
.transcripts/transcript_<timestamp>.jsonl
```

然后调用模型生成摘要，最后把历史替换为一条 user message：

```txt
This conversation was compacted so the agent can continue working.

<summary>
```

摘要会保留：

- 当前目标
- 重要发现和决策
- 读过或改过的文件
- 剩余工作
- 用户约束和偏好

### 手动完整压缩

父代理新增 compact 工具。模型判断对话太长时，可以主动调用：

```ts
compact({ focus: "preserve the current implementation plan" });
```

工具结果先写回历史，随后 agent loop 会触发：

```txt
[manual compact]
```

如果传入 focus，它会追加到摘要里，提醒压缩模型保留下一步最重要的信息。

## 函数具体调用说明

- runOneTurn
    - 微压缩：执行 microCompact，留最后三个 tool 调用的 result
    - 自动完整压缩：判断 messages 长度是否溢出，如果溢出压缩历史，执行 compactHistory
    - 手动完整压缩：模型调用 compact 工具表明需要进行压缩，并且通过 focus 字段说明关注的内容

- compactHistory
    - writeTranscript，把历史写到本地.transcript 目录下的 jsonl 文件去
    - 调用 summarizeHistory 拿到压缩摘要
    - 当 compact 是模型主动调用时，把传入的 focus 内容追加进摘要，让模型知道接下来继续工作需要关注这一点
    - 如果 CompactState.recentFiles 里有最近读过的文件路径，就追加到摘要里
    - 把 summary/focus/recentFiles 合并成新的 summary
    - 更新 state.hasCompacted 和 state.lastSummary
    - 替换整个 messages 历史，只保留一条新的 user message（这里不会把 system prompt 替换掉）

- runRead
    - 新增 trackRecentFile，当文件被读取时被记录下来，方便完整压缩时追加路径
    - 新增 persistLargeOutput，把超长的 tool_result 落到文件，并且返回预览和存储路径
