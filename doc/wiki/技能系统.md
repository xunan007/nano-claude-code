# 技能系统

## 目录约定

技能文件放在工作区的 `skills` 目录下，文件名固定为 `SKILL.md`：

```txt
skills/
  code-review/
    SKILL.md
  docs/
    SKILL.md
```

`SKILL.md` 支持简单 frontmatter：

```md
---
name: code-review
description: Review code changes and prioritize concrete bugs.
---

Full skill instructions...
```

- `name`：工具加载时使用的技能名
- `description`：出现在 system prompt 的轻量目录说明
- 正文：只有调用 `load_skill` 时才进入当前上下文

如果没有 frontmatter，会用父目录名作为技能名，description 默认为 `No description`。

## SkillRegistry

`SkillRegistry` 在进程启动时扫描 `skills/**/SKILL.md`，并维护两层数据：

- `SkillManifest`
    - `name`
    - `description`
    - `path`
- `SkillDocument`
    - `manifest`
    - `body`

关键方法：

- `describeAvailable()`
    - 返回轻量技能目录
    - 注入父代理和子代理的 system prompt
- `loadFullText(name)`
    - 返回 `<skill name="...">...</skill>`
    - 如果技能不存在，会返回已知技能列表

## 工具接入

新增 `load_skill` 工具：

```ts
{
    name: "load_skill",
    description: "Load the full body of a named skill into context.",
    parameters: {
        type: "object",
        properties: { name: { type: "string" } },
        required: ["name"],
    },
}
```

父代理和子代理都可以调用这个工具。这样即使父代理派发了 `task`，子代理在干净上下文里也能根据自己的任务加载需要的专项说明。

## 执行流

```txt
process start
    -> scan skills/**/SKILL.md
    -> system prompt contains only skill catalog
    -> model sees task
    -> if specialized guidance is useful, call load_skill(name)
    -> tool_result returns full skill body
    -> model acts with that loaded guidance
```
