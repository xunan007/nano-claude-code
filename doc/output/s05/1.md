## 📦 项目概览

**nano-claude-code** 是一个基于 Node.js 的 CLI 工具，通过 OpenAI API 提供能力。它使用 **TypeScript** 编写，输出为 ESM 模块，最低要求 Node.js ≥ 20。

---

## 🏗️ 构建与运行

### 开发模式（推荐）
```bash
npm run dev
```
使用 `tsx` 直接执行 `src/index.ts`，无需预先构建，开发迭代最快。

### 生产构建与运行
```bash
npm run build      # tsc -p tsconfig.build.json → dist/
npm run start      # node dist/index.js
```
或者一步到位：
```bash
npm run terminal   # 等价于 build + start
```

### CLI 入口（安装后全局使用）
`bin` 字段指定了 `nano-claude-code` 命令指向 `dist/index.js`，安装后可全局调用。

---

## 📐 TypeScript 配置

- **目标**: ES2022
- **模块系统**: NodeNext（ESM 风格）
- **严格模式**: 开启（`strict`、`noUncheckedIndexedAccess`、`exactOptionalPropertyTypes`）
- **输出**: `dist/`，带 source map、声明文件（`.d.ts`）和声明 map
- **类型检查**（不输出文件）:
  ```bash
  npm run typecheck
  ```

---

## 🧪 代码质量

| 命令 | 工具 | 用途 |
|---|---|---|
| `npm run lint` | ESLint 10 + `typescript-eslint` | 检查 `src/` 下的 TypeScript 代码 |
| `npm run format` | Prettier | 自动格式化所有文件 |
| `npm run format:check` | Prettier | 检查格式是否正确 |
| `npm test` | Vitest | 运行测试 |

ESLint 配置忽略 `dist/`、`node_modules/`、`coverage/`，并为 `no-unused-vars` 设置了 `_` 前缀忽略规则。

## 🔗 关键依赖

| 依赖 | 用途 |
|---|---|
| `openai` | OpenAI API 客户端（核心能力） |

1. **开发** → `npm run dev`
2. **类型检查** → `npm run typecheck`
3. **格式化** → `npm run format`
4. **Lint** → `npm run lint`
5. **测试** → `npm test`
